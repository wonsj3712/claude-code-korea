Claude Code SDK는 애플리케이션에 Claude Code를 임베딩하기 위한 제어 프로토콜입니다. IDE, 자동화 스크립트, CI/CD 파이프라인 등에서 사용할 수 있습니다. 직접적인 라이브러리 API를 노출하는 대신, 실행 중인 `claude` 프로세스와 stdin/stdout을 통해 구조화된 JSON 메시지 스트림으로 통신합니다.

> 호스트 프로세스가 사용자 메시지와 제어 요청을 전송하면, CLI 프로세스가 어시스턴트 메시지, 도구 진행 이벤트, 결과 페이로드를 스트리밍으로 반환합니다.

<Note>
TypeScript 타입은 `@anthropic-ai/claude-code`의 `agentSdkTypes` 엔트리 포인트에서 내보내집니다. 제어 프로토콜 타입(`SDKControl` 접두사)은 `@alpha`로 표시되어 있으며 변경될 수 있습니다.
</Note>

## 작동 방식

### 1단계: Claude Code 프로세스 실행

비대화형 모드를 위한 특정 플래그와 함께 `claude`를 시작합니다:

```bash
claude --output-format stream-json --print --verbose
```

여러 프롬프트를 받는 영속 세션의 경우, `--print`를 생략하고 초기화 후 stdin으로 `SDKUserMessage` 객체를 전송합니다.

### 2단계: 초기화 요청 전송

stdin에 `subtype: "initialize"`가 포함된 `control_request`를 작성합니다. CLI는 사용 가능한 명령어, 모델, 에이전트, 계정 정보가 포함된 `SDKControlInitializeResponse`로 응답합니다.

```json
{
  "type": "control_request",
  "request_id": "init-1",
  "request": {
    "subtype": "initialize",
    "systemPrompt": "You are a code reviewer.",
    "appendSystemPrompt": "Always suggest tests."
  }
}
```

### 3단계: stdout에서 메시지 스트림 처리

stdout에서 줄바꿈으로 구분된 JSON을 읽습니다. 각 줄은 어시스턴트 턴, 도구 진행 상황, 시스템 이벤트, 결과 요약을 포괄하는 하나의 `SDKMessage` 유니온 타입입니다.

### 4단계: 사용자 메시지 전송

stdin에 `SDKUserMessage` 객체를 작성하여 대화를 이어갑니다. 각 메시지에는 Anthropic API 호환 `message` 페이로드가 포함됩니다.

## 출력 형식

| 형식 | 설명 |
|------|------|
| `text` | 일반 텍스트 응답만 출력. 대화형 모드의 기본값. |
| `json` | 완료 시 단일 JSON 객체 출력. 일회성 스크립트에 적합. |
| `stream-json` | 줄바꿈으로 구분된 JSON 스트림. 이벤트 발생 시 한 줄씩 출력. SDK 사용 시 필수. |

점진적 렌더링이나 도구 이벤트 처리에는 `stream-json`을, 최종 결과만 필요한 경우에는 `json`을 사용하세요.

## 제어 프로토콜 메시지

제어 프로토콜은 stdin/stdout을 통해 양방향으로 전달되는 두 가지 최상위 엔벨로프 타입을 사용합니다.

### SDKControlRequest

세션을 구성하거나 명령을 실행하기 위해 CLI 프로세스**로** 전송합니다.

```json
{
  "type": "control_request",
  "request_id": "<unique-string>",
  "request": { "subtype": "...", ...payload }
}
```

**필드:**
- `type` (리터럴: 'control_request') — 항상 `"control_request"`.
- `request_id` (string) — 이 요청의 고유 식별자. CLI가 대응하는 `control_response`에 이 값을 에코합니다.
- `request` (SDKControlRequestInner) — 요청 페이로드. `subtype`으로 수행할 작업을 식별합니다.

### SDKControlResponse

`control_request`에 대한 응답으로 CLI 프로세스**에서** 출력됩니다.

```json
{
  "type": "control_response",
  "response": {
    "subtype": "success",
    "request_id": "<echoed-id>",
    "response": { ...payload }
  }
}
```

오류 발생 시 `subtype`은 `"error"`이며, `error` 필드에 사람이 읽을 수 있는 메시지가 포함됩니다.

## 초기화 요청과 응답

`initialize` 요청은 반드시 가장 먼저 전송해야 하는 제어 메시지입니다. 세션을 구성하고 사용 가능한 기능을 반환합니다.

### SDKControlInitializeRequest

```json
{
  "type": "control_request",
  "request_id": "init-1",
  "request": {
    "subtype": "initialize",
    "systemPrompt": "You are a CI automation agent.",
    "appendSystemPrompt": "Always add test coverage.",
    "hooks": {
      "PreToolUse": [
        {
          "matcher": "Bash",
          "hookCallbackIds": ["my-hook-id"]
        }
      ]
    },
    "agents": {
      "CodeReviewer": {
        "description": "Reviews code for quality and security.",
        "prompt": "You are an expert code reviewer...",
        "model": "opus"
      }
    }
  }
}
```

**필드:**
- `subtype` (리터럴: 'initialize') — 초기화 요청임을 식별합니다.
- `systemPrompt` (string) — 이 세션의 기본 시스템 프롬프트를 대체합니다.
- `appendSystemPrompt` (string) — 기본 시스템 프롬프트를 대체하지 않고 뒤에 추가합니다. 기본 동작을 유지하면서 컨텍스트를 추가할 때 사용합니다.
- `hooks` (Record&lt;HookEvent, SDKHookCallbackMatcher[]&gt;) — SDK 측 훅 콜백을 등록합니다. 훅 이벤트가 발생하면 CLI가 SDK 프로세스로 콜백합니다.
- `sdkMcpServers` (string[]) — 이 세션에 연결할 프로세스 내 SDK MCP 서버(`createSdkMcpServer`로 생성)의 이름 목록.
- `agents` (Record&lt;string, AgentDefinition&gt;) — 이 세션에서 `Agent` 도구가 사용할 수 있는 사용자 정의 서브에이전트 정의.

**AgentDefinition 필드:**
- `description` (string, 필수) — Claude가 이 에이전트를 호출해야 하는 시점에 대한 자연어 설명.
- `prompt` (string, 필수) — 이 에이전트의 시스템 프롬프트.
- `model` (string) — 모델 별칭(`"sonnet"`, `"opus"`, `"haiku"`) 또는 전체 모델 ID. 기본값은 부모 모델 상속.
- `tools` (string[]) — 허용된 도구 이름 목록. 생략 시 모든 도구를 상속합니다.
- `disallowedTools` (string[]) — 이 에이전트에서 명시적으로 차단할 도구 목록.
- `maxTurns` (number) — 에이전트가 중지되기 전 최대 에이전틱 턴 수.
- `permissionMode` (PermissionMode) — 이 에이전트의 권한 모드를 재정의합니다.

### SDKControlInitializeResponse

CLI가 세션의 현재 기능 정보로 응답합니다.

```json
{
  "type": "control_response",
  "response": {
    "subtype": "success",
    "request_id": "init-1",
    "response": {
      "commands": [...],
      "agents": [...],
      "output_style": "stream-json",
      "available_output_styles": ["text", "json", "stream-json"],
      "models": [...],
      "account": {
        "email": "user@example.com",
        "organization": "Acme Corp",
        "apiProvider": "firstParty"
      }
    }
  }
}
```

**필드:**
- `commands` (SlashCommand[]) — 사용 가능한 슬래시 명령어(예: `/compact`, `/cost`). 각 항목에 `name`, `description`, `argumentHint`가 포함됩니다.
- `agents` (AgentInfo[]) — 사용 가능한 서브에이전트 유형. 각각 `name`, `description`, 선택적 `model`이 포함됩니다.
- `output_style` (string) — 활성 출력 형식(`"stream-json"`, `"json"`, `"text"`).
- `models` (ModelInfo[]) — 이 계정에서 사용 가능한 모델 목록.

**ModelInfo 필드:**
- `value` (string) — API 호출용 모델 식별자(예: `"claude-sonnet-4-6"`).
- `displayName` (string) — 사람이 읽을 수 있는 이름(예: `"Claude Sonnet 4.6"`).
- `supportsEffort` (boolean) — 이 모델이 노력 수준(effort levels)을 지원하는지 여부.
- `supportsAdaptiveThinking` (boolean) — 이 모델이 적응형 사고(adaptive thinking)를 지원하는지 여부.

**AccountInfo 필드:**
- `email` (string) — 계정 이메일 주소.
- `organization` (string) — 조직명.
- `subscriptionType` (string) — 구독 티어.
- `apiProvider` ('firstParty' | 'bedrock' | 'vertex' | 'foundry') — 활성 API 백엔드. Anthropic OAuth는 `"firstParty"`일 때만 적용됩니다.

## 사용자 메시지

대화를 진행하기 위해 stdin으로 사용자 메시지를 전송합니다.

### SDKUserMessage

```json
{
  "type": "user",
  "message": {
    "role": "user",
    "content": "Refactor this function to use async/await."
  },
  "parent_tool_use_id": null
}
```

**필드:**
- `type` (리터럴: 'user') — 항상 `"user"`.
- `message` (APIUserMessage) — Anthropic API 호환 사용자 메시지. `content`는 문자열 또는 콘텐츠 블록 배열(이미지 및 기타 미디어용)이 될 수 있습니다.
- `parent_tool_use_id` (string | null) — 이 메시지가 응답하는 도구 사용 ID, 또는 최상위 사용자 메시지인 경우 `null`.
- `uuid` (string) — 이 메시지를 추적하기 위한 선택적 UUID. 관련 이벤트에 에코됩니다.
- `priority` ('now' | 'next' | 'later') — 비동기 메시지 큐잉을 위한 스케줄링 힌트.

## SDK 메시지 스트림 유형

Claude Code는 stdout으로 JSON 메시지 스트림을 출력합니다. `type` 필드로 각 메시지를 식별합니다.

### system - 세션 초기화

세션 시작 시 `subtype: "init"`으로 한 번 출력됩니다. 활성 모델, 도구 목록, MCP 서버 상태, 권한 모드, 세션 ID가 포함됩니다.

```json
{
  "type": "system",
  "subtype": "init",
  "model": "claude-sonnet-4-6",
  "tools": ["Bash", "Read", "Write", "Edit", "Glob", "Grep"],
  "mcp_servers": [],
  "permissionMode": "default",
  "session_id": "abc123",
  "uuid": "..."
}
```

### assistant - 모델 응답

모델이 턴을 생성할 때 출력됩니다. `tool_use` 블록을 포함한 전체 Anthropic API 응답 객체가 포함됩니다.

```json
{
  "type": "assistant",
  "message": { "role": "assistant", "content": [...] },
  "parent_tool_use_id": null,
  "uuid": "...",
  "session_id": "abc123"
}
```

### stream_event - 부분 스트리밍 토큰

스트리밍 중 `RawMessageStreamEvent` 페이로드와 함께 출력됩니다. 점진적 출력 렌더링에 사용합니다.

### tool_progress - 장시간 실행 도구 상태

몇 초 이상 걸리는 도구(예: Bash 명령)에 대해 주기적으로 출력됩니다. `tool_name`, `tool_use_id`, 경과 시간이 포함됩니다.

### result - 최종 턴 요약

각 턴의 끝에 출력됩니다. `subtype`은 `"success"` 또는 오류 하위 유형 중 하나입니다.

```json
{
  "type": "result",
  "subtype": "success",
  "result": "The function has been refactored.",
  "duration_ms": 4200,
  "total_cost_usd": 0.0042,
  "num_turns": 3,
  "is_error": false,
  "stop_reason": "end_turn",
  "session_id": "abc123",
  "uuid": "..."
}
```

오류 하위 유형: `"error_during_execution"`, `"error_max_turns"`, `"error_max_budget_usd"`, `"error_max_structured_output_retries"`.

### system - 상태 업데이트

권한 모드나 세션 상태가 변경될 때(예: `"compacting"`) `subtype: "status"`로 출력됩니다.

## 기타 제어 요청

| `subtype` | 방향 | 설명 |
|-----------|------|------|
| `interrupt` | 호스트 → CLI | 현재 턴을 중단합니다. |
| `set_permission_mode` | 호스트 → CLI | 활성 권한 모드를 변경합니다. |
| `set_model` | 호스트 → CLI | 세션 중간에 다른 모델로 전환합니다. |
| `can_use_tool` | CLI → 호스트 | 도구 호출에 대한 권한 요청 (SDK 권한 핸들러 필요). |
| `mcp_status` | 호스트 → CLI | MCP 서버 연결 상태를 조회합니다. |
| `mcp_set_servers` | 호스트 → CLI | 동적으로 관리되는 MCP 서버를 교체합니다. |
| `get_context_usage` | 호스트 → CLI | 컨텍스트 윈도우 사용량 분석을 조회합니다. |
| `get_settings` | 호스트 → CLI | 병합된 유효 설정을 읽습니다. |
| `apply_flag_settings` | 호스트 → CLI | 플래그 설정 레이어에 설정을 병합합니다. |
| `rewind_files` | 호스트 → CLI | 지정된 메시지 이후의 파일 변경을 되돌립니다. |
| `hook_callback` | CLI → 호스트 | SDK에 등록된 훅 콜백으로 훅 이벤트를 전달합니다. |
| `reload_plugins` | 호스트 → CLI | 디스크에서 플러그인을 다시 로드합니다. |

## 세션 관리 API

스크립팅 시나리오를 위해 SDK는 `~/.claude/`에 저장된 세션 트랜스크립트에 대해 동작하는 함수를 내보냅니다.

```typescript
import {
  query,
  listSessions,
  getSessionInfo,
  getSessionMessages,
  forkSession,
  renameSession,
  tagSession,
} from '@anthropic-ai/claude-code'
```

### query - 프롬프트 실행

SDK의 기본 진입점입니다. `prompt` 문자열 또는 `AsyncIterable<SDKUserMessage>`를 받아 `SDKMessage`의 비동기 이터러블을 반환합니다.

```typescript
for await (const message of query({
  prompt: 'What files are in this directory?',
  options: { cwd: '/my/project' }
})) {
  if (message.type === 'result') {
    console.log(message.result)
  }
}
```

### listSessions - 저장된 세션 목록 조회

프로젝트 디렉토리의 세션 메타데이터를 반환합니다. `dir`을 전달하여 특정 프로젝트로 범위를 지정하거나, 생략하여 모든 세션을 조회합니다.

```typescript
const sessions = await listSessions({ dir: '/my/project', limit: 50 })
```

### getSessionMessages - 트랜스크립트 읽기

세션의 JSONL 트랜스크립트 파일을 파싱하고 시간순으로 메시지를 반환합니다.

```typescript
const messages = await getSessionMessages(sessionId, {
  dir: '/my/project',
  includeSystemMessages: false,
})
```

### forkSession - 대화 분기

세션의 트랜스크립트를 UUID를 재매핑하여 새 세션으로 복사합니다. `upToMessageId`를 사용하여 특정 지점에서 분기할 수 있습니다.

```typescript
const { sessionId: newId } = await forkSession(originalSessionId, {
  upToMessageId: 'msg-uuid',
  title: 'Experimental branch',
})
```

### renameSession과 tagSession

```typescript
await renameSession(sessionId, 'My refactor session')
await tagSession(sessionId, 'needs-review')
await tagSession(sessionId, null) // 태그 제거
```

## 활용 사례

### IDE 통합

IDE는 영속적인 Claude Code 프로세스를 생성하고 제어 프로토콜을 통해 메시지를 라우팅할 수 있습니다. IDE 컨텍스트를 설명하는 사용자 정의 `systemPrompt`와 함께 `initialize` 요청을 전송한 다음, 에디터의 채팅 패널에서 사용자 메시지를 전달합니다. `PreToolUse` 훅 콜백을 사용하여 파일 편집을 가로채고 적용 전에 IDE의 네이티브 UI에 diff를 표시할 수 있습니다.

### CI/CD 자동화

CI 파이프라인에서는 일회성 작업을 위해 `--output-format json`과 `--print`를 사용합니다:

```bash
result=$(echo "Review the diff and output pass/fail" | \
  claude --output-format json --print \
         --permission-mode bypassPermissions)
```

JSON 출력의 `result` 필드를 파싱하여 에이전트의 응답을 프로그래밍 방식으로 추출합니다.

### 헤드리스 에이전트

장시간 실행되는 백그라운드 에이전트의 경우, TypeScript SDK의 `query()`를 스트리밍 출력 형식과 함께 사용합니다. `watchScheduledTasks`(내부)와 결합하여 claude.ai와의 WebSocket 연결을 유지하는 영속적 데몬 프로세스에서 cron 스케줄로 작업을 실행할 수 있습니다.
