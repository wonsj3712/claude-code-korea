## 개요

Claude Code는 세 가지 파일 작업 도구를 제공합니다: 내용을 조회하는 `Read`, 섹션을 수정하는 `Edit`, 생성하거나 전체 덮어쓰기하는 `Write`입니다.

## Read 도구

**목적:** `cat -n` 형식으로 줄 번호와 함께 파일 내용을 조회합니다.

**파라미터:**
- `file_path` (string, 필수): 파일의 절대 경로
- `offset` (number): 시작 줄 번호 (1부터 시작)
- `limit` (number): 반환할 최대 줄 수 (기본값: 2,000)

**지원되는 파일 유형:**
- 텍스트 파일: 줄 번호가 포함된 원시 텍스트
- 이미지 (PNG, JPG): 시각적 표시
- PDF: 문서 내용 (요청당 최대 20페이지, 10페이지 이상의 파일은 `pages` 파라미터 사용)
- Jupyter 노트북 (.ipynb): 모든 셀, 출력, 코드, 텍스트, 시각화

**주요 동작:**
- 디렉토리 목록 조회 불가 (대신 Bash의 `ls` 사용)
- 캐싱: 변경되지 않은 파일은 캐시된 스텁을 반환하여 중복 읽기를 방지합니다

## Edit 도구

**목적:** 파일 내에서 정확한 문자열 치환을 수행합니다.

**파라미터:**
- `file_path` (string, 필수): 대상 파일 경로
- `old_string` (string, 필수): 찾을 정확한 텍스트 (문자 단위 정확한 일치)
- `new_string` (string, 필수): 대체할 텍스트
- `replace_all` (boolean): `true`로 설정하면 모든 발생 항목을 대체합니다

**중요 규칙:**
- Edit를 사용하기 전에 반드시 `Read`를 먼저 호출해야 합니다
- `replace_all: true` 없이 `old_string`이 여러 번 나타나면 실패합니다
- 들여쓰기가 정확히 일치해야 합니다 (`Read` 출력의 줄 번호 접두사는 제외)
- 정규식 해석 없이 일반 문자열 매칭만 수행합니다

## Write 도구

**목적:** 새 파일을 생성하거나 기존 파일을 완전히 다시 작성합니다.

**파라미터:**
- `file_path` (string, 필수): 대상 파일 경로
- `content` (string, 필수): 전체 파일 내용

**요구 사항:**
- 기존 파일을 덮어쓸 때는 반드시 `Read`를 먼저 호출해야 합니다
- 명시적 지시 없이는 문서 파일(*.md, README.*)을 자체적으로 생성하지 않습니다

## 모범 사례

**Write보다 Edit 우선:** 기존 파일을 수정할 때는 변경 사항만 전송하여 컨텍스트를 절약하는 Edit를 Write보다 우선 사용하세요.

**Edit 사용에 적합한 경우:** 섹션 수정, 파일 전체에서 변수명 변경, 코드 블록 추가/제거

**Write 사용에 적합한 경우:** 새 파일 생성, 완전한 재작성, 완전히 다른 내용으로 교체
