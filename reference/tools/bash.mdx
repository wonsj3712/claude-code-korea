## 개요

Bash 도구는 영속적 세션에서 셸 명령을 실행합니다. 작업 디렉토리는 호출 간에 유지되지만, 셸 상태(변수, 별칭, 함수)는 각 호출마다 사용자의 프로필에서 다시 초기화됩니다.

## 핵심 파라미터

**command** (string, 필수): 실행할 셸 명령입니다. 파이프, 리디렉션, 복합 연산자(`&&`, `||`, `;`) 등 모든 표준 bash 구문을 지원합니다.

**timeout** (number): 실행 제한 시간(밀리초)입니다. 기본값은 120,000ms이며 최대 600,000ms까지 설정할 수 있습니다. 환경 변수를 통해 두 값 모두 오버라이드할 수 있습니다.

**description** (string): UI에 표시되는 사람이 읽을 수 있는 설명입니다.

**run_in_background** (boolean): 명령 끝에 `&`를 붙이지 않고도 백그라운드 실행을 활성화합니다.

## 세션 영속성

작업 디렉토리는 호출 간에 유지되지만, 셸 변수, 별칭, 함수, 디렉토리 변경 사항은 유지되지 않습니다. 각 세션은 사용자의 셸 프로필에서 초기화됩니다.

## 타임아웃 설정

기본 타임아웃: 120,000ms (2분)
최대 타임아웃: 600,000ms (10분)

## 권한 모델

명령은 3단계 평가를 거칩니다:
- **허용(Allow)**: 기존 규칙에 일치하거나 읽기 전용으로 분류된 경우
- **확인(Ask)**: 사용자 승인이 필요한 경우
- **거부(Deny)**: 명시적으로 차단된 경우

`ls`, `cat`, `git log` 같은 읽기 전용 명령은 프롬프트 없이 자동으로 허용됩니다.

보안 제한으로 인해 명령 치환, 프로세스 치환, 의심스러운 리디렉션 및 기타 난독화 패턴이 포함된 명령은 차단됩니다.

## 도구 우선순위

Claude는 파일 작업에 전문화된 도구(Read, Edit, Glob, Grep)를 우선 사용하며, Bash는 목적에 맞는 전용 도구가 없는 작업에만 사용합니다.
