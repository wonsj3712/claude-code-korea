## 개요

Claude Code에서 사용할 수 있는 두 가지 웹 상호작용 도구를 설명합니다: 특정 URL을 분석하는 **WebFetch**와 학습 데이터 이후의 최신 정보를 조회하는 **WebSearch**입니다.

## WebFetch 도구

**목적:** HTML을 마크다운으로 변환하고 보조 모델을 통해 처리하여 웹페이지 콘텐츠를 가져오고 분석합니다.

**필수 파라미터:**
- `url` (string): 완전한 형식의 유효한 URL. HTTP는 자동으로 HTTPS로 업그레이드됩니다
- `prompt` (string): 콘텐츠 추출 또는 분석을 위한 자연어 지시문

**작동 방식:**
이 도구는 3단계 프로세스를 따릅니다: URL을 가져오고, HTML을 마크다운으로 변환한 다음, 마크다운과 프롬프트를 보조 모델에 전달하여 합성합니다.

**주요 특성:**
- 원시 페이지 덤프가 아닌 합성된 답변을 반환합니다
- 큰 콘텐츠는 자동으로 요약될 수 있습니다
- 반복적인 URL 호출을 위한 15분 인메모리 캐시를 포함합니다
- 리디렉트 시 자동 추적 대신 특별 메시지를 반환합니다

**제한 사항:**
- 읽기 전용 기능만 제공합니다
- 인증이 필요한 페이지는 처리할 수 없습니다
- GitHub 리소스의 경우 Bash를 통한 `gh` CLI 사용을 권장합니다

**사용 예시:**
- API 인증 세부 정보 및 필수 헤더 추출
- 변경 로그에서 라이브러리 버전 변경 사항 확인
- 사용 예제가 포함된 문서 요약

## WebSearch 도구

**목적:** 웹을 검색하고 출처 인용이 포함된 합성 요약을 제공합니다.

**사용 가능 지역:** 미국에서만 사용 가능하며, 호환되는 API 제공자(Anthropic, Claude 4.0 이상의 Vertex AI, 또는 Foundry)가 필요합니다.

**파라미터:**
- `query` (string): 최소 2자 이상의 검색어
- `allowed_domains` (array, 선택): 결과를 특정 도메인으로 제한
- `blocked_domains` (array, 선택): 지정된 도메인을 결과에서 제외

**출력 구조:** 결과에는 제목, URL, 모델이 합성한 요약이 포함됩니다.

**인용 요구 사항:** 결과에는 "Sources:" 섹션에 마크다운 하이퍼링크가 반드시 포함되어야 합니다.

**검색 용량:** 각 호출마다 자동 쿼리 개선과 함께 최대 8개의 내부 검색을 수행합니다.

**현재 날짜 인식:** 오래된 결과를 방지하기 위해 현재 연도 정보를 자동으로 포함합니다.
