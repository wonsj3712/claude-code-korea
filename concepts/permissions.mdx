Claude Code의 권한 시스템은 파일 작업, bash 명령, MCP 도구 호출 세 가지 작업 범주에 대해 세분화된 제어를 제공합니다. 사용자는 어떤 작업에 명시적 승인이 필요하고, 어떤 작업이 자동으로 실행되는지를 설정할 수 있습니다.

## 권한 모드

### Default 모드

표준 모드입니다. Claude Code는 각 도구 호출을 평가하고, 부수 효과를 일으킬 수 있는 작업에 대해 확인을 요청합니다. 읽기 전용 작업은 자동으로 승인됩니다.

### AcceptEdits 모드

파일 수정은 자동으로 실행되지만, 셸 명령은 여전히 확인이 필요합니다.

### Plan 모드

Claude를 읽기 전용 작업으로 제한합니다. 쓰기 및 bash 작업은 완전히 차단되어, 사용자가 승인하기 전에 제안된 변경 사항을 검토할 수 있습니다.

### BypassPermissions 모드

모든 권한 검사를 비활성화합니다. 이 모드는 **"Claude가 수행할 작업을 사전에 감사한 자동화된, 완전 스크립팅된 워크플로우"**에서만 사용해야 합니다.

### DontAsk 모드

BypassPermissions와 유사하지만 다른 내부 메커니즘을 사용하며, 스크립팅 시나리오를 위해 설계되었습니다.

### Auto 모드

도구 호출을 대화 컨텍스트에 따라 평가하여 승인 여부를 결정하는 실험적인 분류기 기반 접근 방식입니다.

## 설정 방법

사용자는 다음을 통해 권한을 설정할 수 있습니다:
- CLI 플래그: `claude --permission-mode acceptEdits`
- 세션 중 명령: `/permissions`
- 설정 파일: `settings.json`, `.claude/settings.json`, 또는 `.claude/settings.local.json`

## 규칙 시스템

권한 규칙은 세 가지 구성 요소를 포함합니다: `toolName`(어떤 도구), `ruleContent`(선택적 패턴 매칭), `behavior`(allow/deny/ask). 규칙은 전역 모드보다 먼저 평가됩니다.

허용 목록의 예시에는 `"Bash(git status)"` 및 `"Read(*)"` 등이 포함되며, 거부 목록에는 `"Bash(rm -rf *)"` 등이 포함될 수 있습니다.

## Bash 관련 보호

`&&`, `||`, `;`, 파이프 등 연산자를 사용하는 복합 명령은 하위 명령별로 개별 평가되며, 가장 제한적인 결과가 전체 명령을 차단합니다. 특정 구문은 추가 검사를 받습니다: 프로젝트 디렉토리 외부로의 출력 리디렉션, 작업 트리 외부로의 디렉토리 변경, 인플레이스 sed 편집 등이 이에 해당합니다.

안전 검사는 모드에 관계없이 활성 상태를 유지하며, `.claude/` 또는 `.git/` 디렉토리와 셸 설정 파일을 대상으로 하는 명령을 차단합니다.

## MCP 도구 권한

MCP 도구도 동일한 규칙 시스템을 따릅니다. 사용자는 `mcp__servername`으로 전체 서버를 차단하거나, `mcp__servername__tool_name`으로 개별 도구를 차단할 수 있습니다.

## 안전 권장사항

- Default 모드로 시작하세요
- 익숙하지 않은 코드베이스에서는 Plan 모드를 사용하세요
- 광범위한 권한 상승보다 세분화된 허용 규칙을 선호하세요
- 익숙하지 않은 저장소를 사용하기 전에 프로젝트 설정 파일을 검토하세요
