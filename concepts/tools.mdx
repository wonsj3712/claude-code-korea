Claude Code는 Claude에게 사용자의 머신과 상호작용할 수 있는 내장 도구 세트를 제공합니다. 각 도구 호출은 활성 권한 모드와 규칙의 적용을 받습니다. 아래는 기본적으로 사용 가능한 모든 도구에 대한 레퍼런스입니다.

## 파일 도구

### Read

**로컬 파일 시스템에서 파일을 읽습니다.**

기본적으로 최대 2,000줄을 읽습니다. 대용량 파일의 특정 범위 읽기를 위해 `offset`과 `limit`를 지원합니다. `cat -n` 형식으로 줄 번호와 함께 내용을 반환합니다.

이미지(PNG, JPG 등)를 시각적 입력으로 읽기, PDF 파일(한 번에 최대 20페이지), Jupyter 노트북(`.ipynb`)도 지원합니다.

읽기 전용입니다. `default` 모드에서 항상 자동 승인됩니다.

### Edit

**파일에서 정확한 문자열 교체를 수행합니다.**

같은 대화에서 해당 파일에 대한 사전 `Read` 호출이 필요합니다. `old_string`을 `new_string`으로 교체하며 — 파일 내에서 매치가 고유해야 합니다. 파일 전체에서 일괄 교체하려면 `replace_all: true`를 사용하세요.

`replace_all`이 설정되지 않은 상태에서 `old_string`이 두 번 이상 나타나면 실패합니다. 이러한 정밀성은 의도하지 않은 편집을 방지합니다.

### Write

**새 파일을 생성하거나 기존 파일을 완전히 덮어씁니다.**

기존 파일의 경우, 같은 대화에서 사전 `Read` 호출이 필요합니다. 기존 파일을 수정할 때는 `Edit`을 선호하세요 — `Write`는 전체 파일 내용을 전송하므로, 새 파일이나 전체 재작성에 더 적합합니다.

### Glob

**이름 패턴으로 파일을 찾습니다.**

모든 코드베이스 크기에서 작동하는 빠른 패턴 매칭입니다. 수정 시간순으로 정렬된 일치하는 파일 경로를 반환합니다(가장 최근 수정된 것이 먼저).

`**/*.ts`, `src/**/*.test.js`, `**/CLAUDE.md` 같은 패턴을 지원합니다. 개방형 다단계 검색에는 `Task` 도구를 대신 사용하세요.

읽기 전용입니다. 항상 자동 승인됩니다.

## 셸 도구

### Bash

**영속적인 셸 세션에서 셸 명령을 실행합니다.**

대화 내 도구 호출 간에 유지되는 셸에서 명령을 실행합니다 — 환경 변수와 작업 디렉토리 변경이 호출 간에 유지됩니다. `timeout` 파라미터를 지원합니다(기본값 및 최대값은 설정 가능).

주요 동작:

* **복합 명령** (`&&`, `||`, `;`, `|`)은 파싱되어 각 하위 명령이 독립적으로 권한 검사를 받습니다.
* **백그라운드 실행** — 장시간 실행되는 명령을 차단 없이 실행하려면 `run_in_background: true`를 전달하세요. 완료되면 알림을 받습니다.
* **출력 제한** — stdout/stderr가 도구별 결과 크기 예산을 초과하면 잘립니다. 대신 미리보기와 파일 경로가 반환됩니다.
* **검색 명령** (`find`, `grep`, `rg`) — 내용 검색에는 최적화된 권한과 접근을 제공하는 전용 `Grep` 도구를 선호하세요.

`default` 모드에서 권한 확인이 필요합니다. `acceptEdits` 모드에서는 허용 규칙으로 커버된 명령에 대해서만 자동 승인됩니다.

## 검색 도구

### Grep

**정규 표현식을 사용하여 파일 내용을 검색합니다.**

ripgrep 기반입니다. 전체 정규식 문법(`log.*Error`, `function\s+\w+`), 파일 유형 필터링(`*.ts`, `**/*.py`), 세 가지 출력 모드를 지원합니다:

* `files_with_matches` (기본값) — 파일 경로만 반환
* `content` — 컨텍스트와 함께 일치하는 줄 반환
* `count` — 파일별 매치 수 반환

`multiline: true`로 멀티라인 패턴을 지원합니다.

읽기 전용입니다. 항상 자동 승인됩니다.

### LS

**디렉토리 내용을 나열합니다.**

구조화된 형식으로 파일과 하위 디렉토리를 반환합니다. 파일을 읽거나 편집하기 전에 프로젝트 구조를 탐색하는 데 유용합니다.

읽기 전용입니다. 항상 자동 승인됩니다.

최소 모드(`CLAUDE_CODE_SIMPLE=1`)에서는 `Bash`, `Read`, `Edit`만 사용 가능합니다 — 대신 `Bash`를 통해 `ls`를 사용하세요.

## 웹 도구

### WebFetch

**URL을 가져오고 정보를 추출합니다.**

URL과 추출할 내용을 설명하는 프롬프트를 받습니다. HTML을 마크다운으로 변환한 후, 보조 모델을 통해 내용을 처리하여 집중된 답변을 생성합니다.

기능:

* HTTP URL은 자동으로 HTTPS로 업그레이드됩니다.
* 15분 자동 정리 캐시를 포함합니다 — 동일 URL의 반복 요청이 빠릅니다.
* URL이 다른 호스트로 리디렉션될 때, 후속 요청을 위한 리디렉션 URL을 반환합니다.
* GitHub URL의 경우, `Bash`를 통한 `gh` CLI를 선호하세요 (예: `gh pr view`, `gh api`).

읽기 전용입니다. `default` 모드에서 승인을 요청합니다.

### WebSearch

**웹을 검색하고 결과를 반환합니다.**

제목, 스니펫, 마크다운 링크로 포맷된 URL과 함께 검색 결과를 반환합니다. 모델의 학습 데이터 이후 정보에 접근하는 데 유용합니다.

답변 후, Claude는 참조된 모든 URL을 나열하는 `Sources:` 섹션을 자동으로 추가합니다.

특정 사이트를 포함하거나 제외하는 도메인 필터링을 지원합니다. 현재 미국에서만 사용 가능합니다.

`default` 모드에서 승인을 요청합니다.

## 에이전트 및 작업 도구

### Task (Agent)

**작업을 완료하기 위한 서브 에이전트를 생성합니다.**

별도의 컨텍스트에서 중첩된 에이전틱 루프를 시작합니다. 서브 에이전트는 자체 대화 히스토리, 자체 도구 세트(선택적으로 제한 가능)를 가지며, 완료될 때까지 실행한 후 부모 에이전트에 결과를 반환합니다.

서브 에이전트는 다음에서 실행될 수 있습니다:

* **로컬** — 인프로세스, 부모의 파일 시스템과 셸을 공유
* **원격** — 원격 에이전트 적격 기준이 충족될 때 별도의 컴퓨팅에서 실행

개방형 다단계 검색, 병렬 워크스트림, 또는 격리된 에이전트에 별도의 하위 문제를 위임할 때 `Task`를 사용하세요.

부모 에이전트는 서브 에이전트의 최종 출력을 도구 결과로 수신합니다.

### TodoWrite

**구조화된 작업 목록을 생성하고 관리합니다.**

상태(`pending`, `in_progress`, `completed`)가 있는 할 일 항목 목록을 터미널 UI의 영속적 패널에 작성합니다. Claude가 복잡한 다단계 작업의 진행 상황을 추적하고, 현재 작업 중인 내용을 사용자에게 보여줍니다.

3개 이상의 구분된 단계가 있는 작업에 적극적으로 사용하세요. 단순한 단일 단계 요청에는 필요하지 않습니다.

결과는 대화 트랜스크립트가 아닌 할 일 패널에 렌더링됩니다.

## MCP 도구

MCP(Model Context Protocol) 서버는 Claude Code에 추가 도구를 노출할 수 있습니다. 연결된 도구는 내장 도구와 함께 도구 목록에 표시되며, 동일한 권한 시스템을 따릅니다.

MCP 도구는 `mcp__` 접두사로 이름이 지정됩니다:

```
mcp__<server-name>__<tool-name>
```

예를 들어, `mydb`라는 서버의 `query`라는 도구는 `mcp__mydb__query`로 표시됩니다.

일반적인 MCP 도구 카테고리:

* 데이터베이스 쿼리 및 관리 도구
* 브라우저 및 웹 자동화 도구
* 클라우드 제공자 API (AWS, GCP, Azure)
* 이슈 트래커 연동 (GitHub, Linear, Jira)
* 내부 회사 도구 및 API

MCP 서버는 `~/.claude/mcp_servers.json` 또는 `.claude/mcp_servers.json`에서 설정합니다. 연결된 서버의 도구는 서버가 실행 중이면 자동으로 Claude의 컨텍스트에 포함됩니다.

특정 MCP 서버의 모든 도구를 거부하려면 서버 접두사에 대한 거부 규칙을 추가하세요:

```json
{
  "permissions": {
    "deny": ["mcp__untrusted-server"]
  }
}
```

## 노트북 도구

### NotebookEdit

**Jupyter 노트북의 셀을 편집합니다.**

Claude가 줄 수준의 정밀도로 `.ipynb` 파일의 셀을 삽입, 교체 또는 삭제할 수 있게 합니다. 노트북 읽기는 모든 셀과 출력을 반환하는 표준 `Read` 도구를 사용하세요.

## 도구 사용 가능 여부

모든 도구가 모든 컨텍스트에서 사용 가능한 것은 아닙니다. 활성 도구 세트는 시작 시 결정되며, 다음에 의해 영향을 받을 수 있습니다:

* `CLAUDE_CODE_SIMPLE=1` — `Bash`, `Read`, `Edit`로만 제한
* 권한 거부 규칙 — 규칙에 의해 전면 거부된 도구는 모델이 보기 전에 도구 목록에서 제거됩니다
* `isEnabled()` 검사 — 각 도구는 환경 조건에 따라 스스로를 비활성화할 수 있습니다 (예: `WebSearch`는 지역에 의해 제한됨)
* MCP 서버 연결 상태 — MCP 도구는 서버가 실행 중이고 연결된 경우에만 사용 가능합니다

REPL에서 `/tools` 명령으로 활성 도구 세트를 확인할 수 있습니다.
