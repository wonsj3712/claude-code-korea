---

Claude Code는 시작 시 환경 변수를 읽어 설정 파일을 수정하지 않고도 인증, API 접근, 동작 및 런타임 옵션을 구성합니다.

## 인증

### ANTHROPIC_API_KEY

**타입:** `string`

Anthropic API에 직접 인증하기 위한 API 키입니다. OAuth보다 우선 적용됩니다.

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

### ANTHROPIC_AUTH_TOKEN

**타입:** `string`

API_KEY가 적용되지 않는 상황에서 사용하는 대체 인증 토큰입니다.

### ANTHROPIC_BASE_URL

**타입:** `string`

프록시나 스테이징 환경을 위해 Anthropic API 기본 URL을 재정의합니다.

```bash
export ANTHROPIC_BASE_URL="https://my-proxy.example.com"
```

### CLAUDE_CODE_API_BASE_URL

**타입:** `string`

Claude Code 전용 API 기본 URL 재정의입니다. `ANTHROPIC_BASE_URL`보다 우선 적용됩니다.

### ANTHROPIC_BEDROCK_BASE_URL

**타입:** `string`

AWS Bedrock API 접근을 위한 기본 URL입니다.

### ANTHROPIC_VERTEX_PROJECT_ID

**타입:** `string`

Vertex AI 접근을 위한 Google Cloud 프로젝트 ID입니다.

### CLAUDE_CODE_USE_BEDROCK

**타입:** `string`

`1` 또는 `true`로 설정하면 AWS Bedrock을 API 제공자로 사용합니다.

### CLAUDE_CODE_USE_FOUNDRY

**타입:** `string`

`1` 또는 `true`로 설정하면 Anthropic Foundry를 API 제공자로 사용합니다.

### CLAUDE_CODE_OAUTH_TOKEN

**타입:** `string`

자동화 환경에서 대화형 로그인을 건너뛰는 OAuth 액세스 토큰입니다.

## 구성 경로

### CLAUDE_CONFIG_DIR

**타입:** `string` | **기본값:** `~/.claude`

구성, 설정, 트랜스크립트를 저장하는 디렉토리를 재정의합니다.

```bash
export CLAUDE_CONFIG_DIR="/opt/claude-config"
```

### CLAUDE_CODE_MANAGED_SETTINGS_PATH

**타입:** `string`

엔터프라이즈 환경에서 관리형 설정 파일 경로를 재정의합니다.

## 모델 선택

### ANTHROPIC_MODEL

**타입:** `string`

세션의 기본 모델입니다. 설정 파일과 `--model` 플래그로 재정의할 수 있습니다.

### CLAUDE_CODE_SUBAGENT_MODEL

**타입:** `string`

서브 에이전트 작업에 사용할 모델입니다. 설정하지 않으면 메인 세션 모델을 사용합니다.

### CLAUDE_CODE_AUTO_MODE_MODEL

**타입:** `string`

자동 모드 실행에 사용할 모델입니다. 기본값은 메인 세션 모델입니다.

## 동작 토글

### CLAUDE_CODE_REMOTE

**타입:** `string`

`1` 또는 `true`로 설정하면 타임아웃과 서식이 조정된 원격/컨테이너 모드로 전환됩니다.

```bash
export CLAUDE_CODE_REMOTE=1
```

### CLAUDE_CODE_SIMPLE

**타입:** `string`

`1` 또는 `true`로 설정하면 (또는 `--bare` 전달) hooks, LSP, 플러그인, 자격 증명 읽기를 건너뛰는 베어 모드로 전환됩니다. `ANTHROPIC_API_KEY` 인증이 필요합니다.

### DISABLE_AUTO_COMPACT

**타입:** `string`

`1` 또는 `true`로 설정하면 자동 컨텍스트 압축을 비활성화합니다.

```bash
export DISABLE_AUTO_COMPACT=1
```

### CLAUDE_CODE_DISABLE_BACKGROUND_TASKS

**타입:** `boolean`

`1` 또는 `true`로 설정하면 백그라운드 셸 명령 실행을 비활성화합니다.

### CLAUDE_CODE_DISABLE_THINKING

**타입:** `string`

`1` 또는 `true`로 설정하면 모든 API 호출에서 확장 사고(extended thinking)를 비활성화합니다.

### CLAUDE_CODE_DISABLE_AUTO_MEMORY

**타입:** `string`

`1` 또는 `true`로 설정하면 자동 메모리를 비활성화합니다. `0` 또는 `false`로 설정하면 명시적으로 활성화합니다. 베어 모드와 원격 모드에서는 명시적으로 활성화하지 않는 한 자동으로 비활성화됩니다.

### CLAUDE_CODE_DISABLE_CLAUDE_MDS

**타입:** `string`

`1` 또는 `true`로 설정하면 모든 CLAUDE.md 메모리 파일 로딩을 비활성화합니다.

### CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC

**타입:** `string`

`1` 또는 `true`로 설정하면 분석 및 텔레메트리를 억제합니다.

### CLAUDE_CODE_ADDITIONAL_DIRECTORIES_CLAUDE_MD

**타입:** `string`

`1` 또는 `true`로 설정하면 `--add-dir`로 추가된 디렉토리에서 CLAUDE.md 파일을 로드합니다.

### CLAUDE_BASH_MAINTAIN_PROJECT_WORKING_DIR

**타입:** `string`

`1` 또는 `true`로 설정하면 각 Bash 명령 실행 후 작업 디렉토리를 프로젝트 루트로 재설정합니다.

## 리소스 제한

### CLAUDE_CODE_MAX_OUTPUT_TOKENS

**타입:** `number`

API 응답당 최대 출력 토큰 수를 재정의합니다.

```bash
export CLAUDE_CODE_MAX_OUTPUT_TOKENS=4096
```

### CLAUDE_CODE_MAX_CONTEXT_TOKENS

**타입:** `number`

모델의 기본값 대신 최대 컨텍스트 윈도우 크기를 재정의합니다.

### BASH_MAX_OUTPUT_LENGTH

**타입:** `number`

Bash 출력에서 캡처하는 최대 문자 수입니다. 초과 시 잘립니다.

```bash
export BASH_MAX_OUTPUT_LENGTH=50000
```

### API_TIMEOUT_MS

**타입:** `number`

API 요청 타임아웃을 밀리초 단위로 재정의합니다. 기본값은 표준 모드에서 300,000ms, 원격 모드에서 120,000ms입니다.

```bash
export API_TIMEOUT_MS=60000
```

## 텔레메트리 및 관측 가능성

### CLAUDE_CODE_ENABLE_TELEMETRY

**타입:** `string`

`1` 또는 `true`로 설정하면 OpenTelemetry 내보내기를 활성화합니다. `OTEL_EXPORTER_OTLP_ENDPOINT` 구성이 필요합니다.

```bash
export CLAUDE_CODE_ENABLE_TELEMETRY=1
export OTEL_EXPORTER_OTLP_ENDPOINT="http://localhost:4318"
```

### CLAUDE_CODE_JSONL_TRANSCRIPT

**타입:** `string`

JSONL 세션 트랜스크립트를 출력할 파일 경로입니다.

```bash
export CLAUDE_CODE_JSONL_TRANSCRIPT="/tmp/session.jsonl"
```

## Node.js 런타임

### NODE_OPTIONS

**타입:** `string`

표준 Node.js 옵션입니다. Claude Code는 `--max-old-space-size` 같은 플래그를 감지합니다.

```bash
export NODE_OPTIONS="--max-old-space-size=4096"
```

<Warning>
코드 실행 플래그는 피하세요. Claude Code는 셸 환경을 검증합니다.
</Warning>

## 호스트 플랫폼 재정의

### CLAUDE_CODE_HOST_PLATFORM

**타입:** `string`

보고되는 호스트 플랫폼을 재정의합니다. `"win32"`, `"darwin"`, `"linux"` 중 하나를 사용합니다. 컨테이너 환경에서 유용합니다.

```bash
export CLAUDE_CODE_HOST_PLATFORM=darwin
```

## 클라우드 제공자 리전 재정의

### Vertex AI 리전 재정의

모델 프리픽스별로 환경 변수를 사용하여 Vertex AI 리전을 재정의할 수 있습니다:

| 모델 프리픽스 | 환경 변수 |
|---|---|
| claude-haiku-4-5 | VERTEX_REGION_CLAUDE_HAIKU_4_5 |
| claude-3-5-haiku | VERTEX_REGION_CLAUDE_3_5_HAIKU |
| claude-3-5-sonnet | VERTEX_REGION_CLAUDE_3_5_SONNET |
| claude-3-7-sonnet | VERTEX_REGION_CLAUDE_3_7_SONNET |
| claude-opus-4-1 | VERTEX_REGION_CLAUDE_4_1_OPUS |
| claude-opus-4 | VERTEX_REGION_CLAUDE_4_0_OPUS |
| claude-sonnet-4-6 | VERTEX_REGION_CLAUDE_4_6_SONNET |
| claude-sonnet-4-5 | VERTEX_REGION_CLAUDE_4_5_SONNET |
| claude-sonnet-4 | VERTEX_REGION_CLAUDE_4_0_SONNET |

```bash
export VERTEX_REGION_CLAUDE_4_0_OPUS="us-central1"
```

기본 Vertex 리전은 `CLOUD_ML_REGION`으로 제어됩니다 (기본값: `us-east5`).

## AWS 자격 증명

### AWS_REGION

**타입:** `string`

Bedrock API 호출을 위한 AWS 리전입니다. `AWS_DEFAULT_REGION`으로 폴백하며, 그것도 없으면 `us-east-1`이 기본값입니다.

### AWS_DEFAULT_REGION

**타입:** `string`

`AWS_REGION`이 설정되지 않았을 때의 폴백 AWS 리전입니다.

## 모든 세션에 환경 변수 설정하기

셸 프로파일 대신 설정 파일의 `env` 필드에 환경 변수를 설정할 수 있습니다. 이렇게 하면 셸을 수정하지 않고 세션별 구성이 가능합니다.

```json
{
  "env": {
    "DISABLE_AUTO_COMPACT": "1",
    "BASH_MAX_OUTPUT_LENGTH": "30000"
  }
}
```

`env` 필드에 대한 자세한 내용은 [설정](/configuration/settings) 문서를 참조하세요.
