---
title: "설정"
description: "사용자, 프로젝트, 관리형 수준의 JSON 설정 파일을 통한 Claude Code 동작 구성"
---

Claude Code는 여러 범위의 JSON 파일에서 설정을 읽습니다. 설정은 가장 낮은 우선순위에서 가장 높은 우선순위 순으로 병합되므로, 더 구체적인 범위가 더 넓은 범위를 재정의합니다.

## 설정 파일

### 글로벌 (사용자)

**위치:** `~/.claude/settings.json`

모든 프로젝트에 걸쳐 실행하는 모든 Claude Code 세션에 적용됩니다. 선호하는 모델, 테마, 정리 정책 같은 개인 설정을 여기에 지정합니다.

```json
{
  "model": "claude-opus-4-5",
  "cleanupPeriodDays": 30,
  "permissions": {
    "defaultMode": "acceptEdits"
  }
}
```

### 프로젝트 (공유)

**위치:** 프로젝트 루트의 `.claude/settings.json`

소스 컨트롤에 커밋됩니다. 프로젝트에서 작업하는 모든 사람에게 적용되어야 하는 설정에 사용합니다 — 권한 규칙, hook 구성, MCP 서버, 환경 변수 등.

```json
{
  "permissions": {
    "allow": ["Bash(npm run *)", "Bash(git *)"],
    "deny": ["Bash(rm -rf *)"]
  },
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [{ "type": "command", "command": "npm run lint" }]
      }
    ]
  }
}
```

### 로컬 (개인 프로젝트)

**위치:** 프로젝트 루트의 `.claude/settings.local.json`

소스 컨트롤에 커밋되지 않습니다 (자동으로 `.gitignore`에 추가됨). 특정 프로젝트 내에서의 개인 재정의에 사용합니다 — 공유하지 않아야 할 권한 설정이나 환경 변수 등.

```json
{
  "permissions": {
    "defaultMode": "bypassPermissions"
  }
}
```

### 관리형 (엔터프라이즈)

**위치:** 플랫폼별 시스템 경로

관리자가 MDM, 레지스트리 (Windows), plist (macOS) 또는 관리형 설정 파일을 통해 설정합니다. 관리형 설정은 가장 높은 우선순위를 가지며 사용자나 프로젝트가 재정의할 수 없습니다.

## 설정 열기

Claude Code 세션에서 `/config`를 실행하면 설정 UI가 열립니다. 설정 패널의 **Config** 탭이 열리며, 각 범위별로 현재 활성화된 설정을 찾아보고 편집할 수 있습니다.

JSON 파일을 직접 편집할 수도 있습니다. Claude Code는 파일 변경을 감지하면 자동으로 설정을 다시 로드합니다.

## 설정 우선순위

설정은 가장 낮은 우선순위에서 가장 높은 우선순위 순으로 병합됩니다. 스칼라 값의 경우 나중 소스가 이전 소스를 재정의하며, 배열은 연결 후 중복이 제거됩니다.

```
플러그인 기본값 → 사용자 설정 → 프로젝트 설정 → 로컬 설정 → 관리형(정책) 설정
```

<Note>
관리형(정책) 설정은 항상 최종 우선순위를 가집니다. 관리자가 `permissions.defaultMode`를 `"default"`로 설정하면 사용자나 프로젝트 파일이 이를 재정의할 수 없습니다.
</Note>

## 설정 레퍼런스

### model

**타입:** `string` | **범위:** 모든 범위

Claude Code에서 사용하는 기본 모델을 재정의합니다. 구성된 제공자가 지원하는 모든 모델 ID를 사용할 수 있습니다.

```json
{ "model": "claude-opus-4-5" }
```

### permissions

**타입:** `object` | **범위:** 모든 범위

Claude가 사용할 수 있는 도구와 모드를 제어합니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `allow` | `string[]` | Claude가 확인 없이 수행할 수 있는 작업 규칙 |
| `deny` | `string[]` | Claude가 항상 차단되는 작업 규칙 |
| `ask` | `string[]` | 항상 확인을 요청하는 작업 규칙 |
| `defaultMode` | `string` | 기본 권한 모드: `"default"`, `"acceptEdits"`, `"plan"`, 또는 `"bypassPermissions"` |
| `disableBypassPermissionsMode` | `"disable"` | 사용자가 권한 우회 모드에 진입하는 것을 방지 |
| `additionalDirectories` | `string[]` | Claude가 접근할 수 있는 추가 디렉토리 |

```json
{
  "permissions": {
    "defaultMode": "acceptEdits",
    "allow": ["Bash(npm run *)", "Bash(git log *)"],
    "deny": ["Bash(curl *)"]
  }
}
```

### hooks

**타입:** `object` | **범위:** 모든 범위

도구 실행 전후에 사용자 정의 셸 명령을 실행합니다.

지원되는 hook 이벤트: `PreToolUse`, `PostToolUse`, `Notification`, `UserPromptSubmit`, `SessionStart`, `SessionEnd`, `Stop`, `SubagentStop`, `PreCompact`, `PostCompact`.

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [{ "type": "command", "command": "prettier --write $CLAUDE_FILE_PATHS" }]
      }
    ]
  }
}
```

### cleanupPeriodDays

**타입:** `integer` (음수 아닌 정수) | **기본값:** `30` | **범위:** 모든 범위

채팅 트랜스크립트를 보관하는 일수입니다. `0`으로 설정하면 세션 지속성이 완전히 비활성화됩니다 — 트랜스크립트가 기록되지 않으며 기존 것들은 시작 시 삭제됩니다.

```json
{ "cleanupPeriodDays": 7 }
```

### env

**타입:** `object` | **범위:** 모든 범위

모든 Claude Code 세션에 주입할 환경 변수입니다. 값은 문자열로 변환됩니다.

```json
{
  "env": {
    "NODE_ENV": "development",
    "MY_API_URL": "https://api.example.com"
  }
}
```

### availableModels

**타입:** `string[]` | **범위:** 관리형

사용자가 선택할 수 있는 모델의 엔터프라이즈 허용 목록입니다. 패밀리 별칭(`"opus"`는 모든 Opus 버전 허용), 버전 프리픽스, 전체 모델 ID를 사용할 수 있습니다. `undefined`이면 모든 모델을 사용할 수 있습니다. 빈 배열이면 기본 모델만 사용할 수 있습니다.

```json
{ "availableModels": ["claude-opus-4-5", "claude-sonnet-4"] }
```

### allowedMcpServers / deniedMcpServers

**타입:** `object[]` | **범위:** 모든 범위

MCP 서버의 엔터프라이즈 허용 목록 및 차단 목록입니다. 각 항목은 `serverName`, `serverCommand`, `serverUrl` 중 정확히 하나를 가져야 합니다. 차단 목록이 우선합니다 — 서버가 두 목록 모두에 일치하면 차단됩니다.

```json
{
  "allowedMcpServers": [
    { "serverName": "my-db-server" },
    { "serverUrl": "https://mcp.example.com/*" }
  ],
  "deniedMcpServers": [
    { "serverCommand": ["npx", "malicious-server"] }
  ]
}
```

### worktree

**타입:** `object` | **범위:** 모든 범위

`--worktree` 플래그 동작을 위한 구성입니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `symlinkDirectories` | `string[]` | 메인 레포에서 worktree로 심볼릭 링크할 디렉토리 (예: `"node_modules"`) |
| `sparsePaths` | `string[]` | 대규모 모노레포에서 더 빠른 worktree를 위해 sparse 모드로 체크아웃할 경로 |

```json
{
  "worktree": {
    "symlinkDirectories": ["node_modules", ".cache"],
    "sparsePaths": ["src/", "packages/my-service/"]
  }
}
```

### attribution

**타입:** `object` | **범위:** 모든 범위

Claude가 커밋과 PR 설명에 추가하는 기여 표시 텍스트를 커스터마이징합니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `commit` | `string` | 기여 표시 텍스트 (git 트레일러 포함). 빈 문자열은 기여 표시를 제거합니다. |
| `pr` | `string` | PR 설명용 기여 표시 텍스트. 빈 문자열은 기여 표시를 제거합니다. |

```json
{
  "attribution": {
    "commit": "Co-Authored-By: Claude <noreply@anthropic.com>",
    "pr": ""
  }
}
```

### language

**타입:** `string` | **범위:** 모든 범위

Claude 응답 및 음성 받아쓰기에 사용할 선호 언어입니다. 일반 언어 이름을 사용합니다.

```json
{ "language": "japanese" }
```

### sandbox

**타입:** `object` | **범위:** 모든 범위

Claude의 도구 실행을 격리하기 위한 샌드박스 구성입니다. 샌드박스 설정에 대한 자세한 내용은 관리자에게 문의하세요.

### alwaysThinkingEnabled

**타입:** `boolean` | **기본값:** `true` | **범위:** 모든 범위

`false`이면 확장 사고(extended thinking)가 비활성화됩니다. 없거나 `true`이면 이를 지원하는 모델에서 자동으로 사고 기능이 활성화됩니다.

```json
{ "alwaysThinkingEnabled": false }
```

### effortLevel

**타입:** `"low" | "medium" | "high"` | **범위:** 모든 범위

조정 가능한 사고 예산을 지원하는 모델의 지속적 노력 수준입니다.

```json
{ "effortLevel": "high" }
```

### autoMemoryEnabled

**타입:** `boolean` | **범위:** 사용자, 로컬

이 프로젝트의 자동 메모리를 활성화하거나 비활성화합니다. `false`이면 Claude가 자동 메모리 디렉토리에서 읽거나 쓰지 않습니다.

```json
{ "autoMemoryEnabled": false }
```

### autoMemoryDirectory

**타입:** `string` | **범위:** 사용자, 로컬

자동 메모리 저장소의 사용자 정의 경로입니다. 홈 디렉토리 확장을 위해 `~/`를 지원합니다. 보안상의 이유로 프로젝트 설정(`.claude/settings.json`)에서 설정하면 무시됩니다. 기본값은 `~/.claude/projects/<sanitized-cwd>/memory/`입니다.

```json
{ "autoMemoryDirectory": "~/my-memory-store" }
```

### claudeMdExcludes

**타입:** `string[]` | **범위:** 모든 범위

로딩에서 제외할 `CLAUDE.md` 파일의 glob 패턴 또는 절대 경로입니다. 패턴은 picomatch를 사용하여 절대 경로와 매칭됩니다. 사용자, 프로젝트, 로컬 메모리 유형에만 적용됩니다 — 관리형 파일은 제외할 수 없습니다.

```json
{
  "claudeMdExcludes": [
    "/home/user/monorepo/vendor/CLAUDE.md",
    "**/third-party/.claude/rules/**"
  ]
}
```

### disableAllHooks

**타입:** `boolean` | **범위:** 모든 범위

`true`로 설정하면 모든 hook 및 `statusLine` 실행을 비활성화합니다.

```json
{ "disableAllHooks": true }
```

### respectGitignore

**타입:** `boolean` | **기본값:** `true` | **범위:** 모든 범위

파일 선택기가 `.gitignore` 파일을 준수할지 여부입니다. `.ignore` 파일은 이 설정과 관계없이 항상 준수됩니다.

```json
{ "respectGitignore": false }
```

### defaultShell

**타입:** `"bash" | "powershell"` | **기본값:** `"bash"` | **범위:** 모든 범위

입력 상자에서 `!` 명령에 사용할 기본 셸입니다.

```json
{ "defaultShell": "powershell" }
```

### forceLoginMethod

**타입:** `"claudeai" | "console"` | **범위:** 관리형

특정 로그인 방법을 강제합니다. Claude Pro/Max 계정은 `"claudeai"`, Anthropic Console 결제는 `"console"`을 사용합니다.

```json
{ "forceLoginMethod": "console" }
```

### apiKeyHelper

**타입:** `string` | **범위:** 사용자, 관리형

인증 값을 출력하는 스크립트의 경로입니다. 정적 API 키를 읽는 대신 동적으로 자격 증명을 가져오기 위해 스크립트가 호출됩니다.

```json
{ "apiKeyHelper": "/usr/local/bin/get-claude-key.sh" }
```

### syntaxHighlightingDisabled

**타입:** `boolean` | **범위:** 모든 범위

diff에서 구문 강조를 비활성화합니다.

```json
{ "syntaxHighlightingDisabled": true }
```

### prefersReducedMotion

**타입:** `boolean` | **범위:** 모든 범위

접근성을 위해 애니메이션(스피너 쉬머, 플래시 효과 등)을 줄이거나 비활성화합니다.

```json
{ "prefersReducedMotion": true }
```

## 관리형 설정 (엔터프라이즈)

관리자는 플랫폼 네이티브 메커니즘을 통해 모든 사용자에게 설정을 배포할 수 있습니다. 관리형 설정은 모든 사용자 및 프로젝트 설정보다 우선합니다.

### macOS

`/Library/Preferences/`에 plist 파일을 배포하거나 MDM (Jamf, Kandji 등)을 통해 `com.anthropic.claudecode`를 대상으로 배포합니다.

### Windows

`HKLM\Software\Anthropic\Claude Code` 레지스트리 키 (시스템 전체) 또는 `HKCU\Software\Anthropic\Claude Code` (사용자별, HKLM보다 낮은 우선순위)에 설정을 기록합니다.

### 파일 기반

플랫폼별 관리형 경로에 `managed-settings.json` 파일을 배치합니다. 드롭인 구성 조각을 위해 같은 위치에 `managed-settings.d/` 디렉토리를 만듭니다. 해당 디렉토리의 파일은 알파벳순으로 정렬되어 기본 파일 위에 병합됩니다 — 나중 파일명이 우선합니다.

```
managed-settings.json          # 기본 설정 (가장 낮은 우선순위)
managed-settings.d/
  10-security.json             # 먼저 병합됨
  20-model-policy.json         # 두 번째로 병합됨 (10-보다 우선)
```

이 규칙은 systemd/sudoers 드롭인과 동일합니다: 독립적인 팀이 단일 파일의 편집을 조율하지 않고도 정책 조각을 배포할 수 있습니다.

다음 관리형 전용 설정은 사용자 커스터마이징 영역을 잠급니다:

| 설정 | 설명 |
|------|------|
| `allowManagedHooksOnly` | `true`이면 관리형 설정에 정의된 hook만 실행됩니다. 사용자, 프로젝트, 로컬 hook은 무시됩니다. |
| `allowManagedPermissionRulesOnly` | `true`이면 관리형 설정의 권한 규칙만 적용됩니다. |
| `allowManagedMcpServersOnly` | `true`이면 `allowedMcpServers` 허용 목록이 관리형 설정에서만 읽힙니다. |
| `strictPluginOnlyCustomization` | 특정 커스터마이징 영역(`"skills"`, `"agents"`, `"hooks"`, `"mcp"`)을 플러그인 전용 소스로 잠급니다. 네 가지 모두 잠그려면 `true`를, 또는 영역 이름의 배열을 전달합니다. |

<Warning>
`allowManagedPermissionRulesOnly`는 사용자 설정, 프로젝트 설정, 로컬 설정, CLI 인수의 권한 규칙을 무시합니다. 이를 활성화하기 전에 관리형 규칙이 포괄적인지 확인하세요.
</Warning>

## JSON 스키마

편집기 자동완성 및 유효성 검사를 위해 설정 파일에 `$schema`를 추가하세요:

```json
{
  "$schema": "https://schemas.anthropic.com/claude-code/settings.json"
}
```
