---

Skills는 재사용 가능한 프롬프트와 워크플로우를 정의하는 마크다운 파일입니다. Claude Code에서 `/skill-name`을 입력하면 Claude가 해당 skill의 지침을 로드하고 설명된 작업을 실행합니다.

## Skills의 작동 방식

Skill은 `SKILL.md` 파일이 있는 `.claude/skills/` 내부의 디렉토리입니다. `/skill-name`을 호출하면 Claude Code가 skill의 `SKILL.md`를 프롬프트로 로드합니다. Skills는 주문형으로 로드되어 시작 성능에 영향을 주지 않으므로, 세션 간에 반복 가능한 워크플로우에 이상적입니다.

## Skill 만들기

<Steps>
  <Step title="디렉토리 생성">
    ```bash
    mkdir -p .claude/skills/my-skill
    ```
  </Step>

  <Step title="SKILL.md 작성">
    frontmatter(설명, 힌트)와 `$ARGUMENTS` 플레이스홀더를 사용한 지침을 포함하여 `SKILL.md`를 작성합니다:

    ```markdown
    ---
    description: 이 프로젝트의 전체 릴리스 프로세스를 실행합니다
    argument-hint: 버전 번호 (예: 1.2.3)
    ---

    $ARGUMENTS 버전으로 프로젝트를 릴리스합니다.

    단계:
    1. `package.json`의 버전을 $ARGUMENTS로 업데이트
    2. 이 버전에 대한 새 섹션으로 CHANGELOG.md를 업데이트
    3. `npm test`를 실행하고 모든 테스트가 통과하는지 확인
    4. "chore: release v$ARGUMENTS" 메시지로 커밋
    5. `v$ARGUMENTS` git 태그를 생성
    ```
  </Step>

  <Step title="슬래시 명령으로 호출">
    ```
    /my-skill 1.2.3
    ```
  </Step>
</Steps>

## Skill Frontmatter

Frontmatter는 선택적 메타데이터 필드를 지원합니다:

```yaml
---
description: /skills에 표시되고 Claude가 사용 시점을 판단하는 데 사용하는 간단한 설명
argument-hint: 인수로 전달할 내용 (자동완성에 표시)
allowed-tools: Bash, Write, Read
when_to_use: 사용자가 새 컴포넌트를 만들도록 요청할 때 이 skill을 사용
model: claude-sonnet-4-6
user-invocable: true
context: fork
---
```

| 필드           | 설명                                                                           |
| -------------- | ------------------------------------------------------------------------------ |
| description    | `/skills`에 표시되고 Claude가 사용 시점을 판단하는 데 사용하는 간단한 설명         |
| argument-hint  | 어떤 인수를 제공해야 하는지에 대한 자동완성 안내                                   |
| allowed-tools  | 쉼표로 구분된 허용 도구 목록 (기본값: 전체)                                       |
| when_to_use    | 사전 호출 시나리오를 설명하는 문구                                                |
| model          | 특정 Claude 모델 오버라이드                                                      |
| user-invocable | 슬래시 명령 목록에서의 표시 여부 토글                                             |
| context        | `fork`로 설정하면 격리된 서브에이전트에서 skill 실행                               |
| paths          | 조건부 활성화를 위한 glob 패턴                                                   |
| version        | 버전 식별 문자열                                                                |
| hooks          | 설정 형식과 일치하는 이벤트 기반 트리거                                            |

## 인수 치환

위치 인수에는 `$ARGUMENTS`를 사용합니다:

```
프로젝트의 규칙을 따라 $ARGUMENTS라는 이름의 새 React 컴포넌트를 생성합니다.
```

이름이 지정된 인수의 경우, frontmatter에서 선언한 후 참조합니다:

```yaml
---
arguments: [name, directory]
---
```

그런 다음 skill 본문에서 `$name`과 `$directory`로 참조합니다.

## 인라인 셸 명령

`!` 접두사 뒤에 백틱으로 감싼 명령은 해당 명령을 실행하고 그 출력으로 블록을 대체합니다. 명령은 호출 시(시작 시가 아님) 실행되며 현재 사용자의 셸 권한으로 실행됩니다.

```markdown
---
description: 최근 변경 사항 검토
---

다음은 컨텍스트를 위한 최근 커밋입니다:

!`git log --oneline -20`

위 변경 사항을 검토하고 수행된 내용을 요약합니다.
```

## Skill 목록 확인

사용 가능한 모든 skills를 보려면 Claude Code에서 `/skills`를 입력합니다.

## 네임스페이스 Skills

중첩된 skills는 콜론으로 구분됩니다:

```
.claude/skills/
  deployment/
    SKILL.md      → /deployment
  database/
    migrate/
      SKILL.md    → /database:migrate
    seed/
      SKILL.md    → /database:seed
```

## 조건부 Skills (경로 기반 활성화)

`paths`를 사용하면 일치하는 파일이 편집될 때만 skill 사용이 가능하도록 제한합니다:

```yaml
---
description: Django 모델 리뷰
paths: "**/*.py"
when_to_use: Django 모델 파일을 편집할 때 사용
---
```

## 번들 Skills

번들 skills는 Claude Code에 미리 컴파일되어 자동으로 제공되는 기능입니다.

## 사용자 수준 Skills

사용자 수준 skills는 `~/.claude/skills/`에 위치하며 모든 프로젝트에서 사용할 수 있습니다.

```bash
mkdir -p ~/.claude/skills/standup
cat > ~/.claude/skills/standup/SKILL.md << 'EOF'
---
description: 오늘 작업한 내용을 스탠드업 업데이트용으로 요약합니다
---

이 저장소에서 오늘의 git 커밋을 확인하고 스탠드업 형식으로 요약합니다: 무엇을 했는지, 다음에 무엇을 할 것인지, 차단 요소가 있는지. 3-4문장으로 작성합니다.
EOF
```

## Skills vs. Hooks

| 기능        | Skills                               | Hooks                                           |
| ----------- | ------------------------------------ | ----------------------------------------------- |
| 호출 방식   | 명시적 `/skill-name` 또는 Claude 인식 | 도구 이벤트 시 자동                               |
| 사용 사례   | 의도적인 반복 가능 워크플로우           | 자동 부수 효과, 린팅, 차단                         |
| 설정 방식   | `.claude/skills/`의 `SKILL.md`       | 설정 JSON의 hooks 필드                            |
| 컨텍스트    | 파일 및 셸 출력 지원                   | 이벤트 JSON 수신                                  |

Skills는 사용자가 의도적으로 트리거하는 명명된 워크플로우입니다. Hooks는 이벤트에 대한 자동 응답을 제공합니다. 사용자가 직접 트리거하는 의도적이고 반복 가능한 작업에는 skills를, 자동 부수 효과에는 hooks를 선택하세요.

## 예제: 커스텀 컴포넌트 생성기

```markdown
---
description: 테스트가 포함된 새 React 컴포넌트를 생성합니다
argument-hint: ComponentName
allowed-tools: Write, Bash
---

$ARGUMENTS라는 이름의 새 React 컴포넌트를 생성합니다.

1. 다음 내용으로 `src/components/$ARGUMENTS/$ARGUMENTS.tsx`를 생성:
   - TypeScript를 사용하는 함수형 컴포넌트
   - `$ARGUMENTSProps`라는 이름의 Props 인터페이스
   - 컴포넌트를 설명하는 JSDoc 주석
   - default export

2. 다음 내용으로 `src/components/$ARGUMENTS/$ARGUMENTS.test.tsx`를 생성:
   - React Testing Library를 사용한 렌더링 테스트 최소 하나
   - 스냅샷 테스트

3. 컴포넌트를 re-export하는 `src/components/$ARGUMENTS/index.ts`를 생성.

4. 타입 오류가 없는지 확인하기 위해 `npx tsc --noEmit`를 실행.
```

호출 예시:

```
/new-component Button
```
