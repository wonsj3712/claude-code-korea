Claude Code는 서브에이전트를 생성할 수 있습니다. 서브에이전트는 독립적으로 실행되어 작업을 병렬로 완료하는 별도의 Claude 인스턴스입니다. 이를 통해 대규모 다단계 작업을 전문화된 에이전트들에게 분할하여 동시에 처리할 수 있습니다.

## 서브에이전트의 작동 방식

Claude가 `Agent` 도구를 배포하면 자체 컨텍스트 윈도우, 전문화된 시스템 프롬프트, 설정 가능한 도구 권한을 갖춘 새로운 Claude 인스턴스가 시작됩니다. 시작한 Claude는 완료를 기다리거나 에이전트가 비동기로 실행되는 경우 다른 작업을 계속하며, 최종적으로 에이전트의 출력을 통합된 응답으로 받습니다.

각 생성된 에이전트의 특징:
- 새로운 컨텍스트 윈도우로 시작합니다 (fork가 아닌 경우 부모 컨텍스트를 상속하지 않음)
- 에이전트 유형에 맞는 전문화된 시스템 프롬프트를 받습니다
- 에이전트 유형별로 설정 가능한 독립적인 도구 권한을 가집니다
- 추가 서브에이전트를 생성할 수 있지만, 중첩에는 제한이 있습니다

## Agent 도구

Claude가 서브에이전트를 실행하기 위해 `Agent` 도구를 호출합니다. 사용자가 직접 호출하는 것이 아니라 Claude가 생성이 적절한 시점을 판단합니다. 이 도구는 다음 파라미터를 받습니다:

- **description**: 에이전트 목표에 대한 3-5단어 요약 (UI에 표시됨)
- **prompt**: 에이전트를 위한 완전한 작업 설명
- **subagent_type**: 전문화된 에이전트 유형 선택 (선택 사항; 기본값은 범용)
- **run_in_background**: 비동기 실행을 위한 불리언 값으로, Claude가 다른 작업을 계속할 수 있게 합니다
- **isolation**: 에이전트에게 격리된 git 워크트리를 제공하려면 `"worktree"`로 설정

## Claude가 서브에이전트를 사용하는 경우

Claude는 작업이 병렬화 또는 전문화의 이점을 얻을 수 있을 때 에이전트를 생성합니다. 여기에는 독립적인 병렬 작업, 전문화된 작업, 장시간 실행 작업, 격리된 탐색이 포함됩니다.

## 멀티 에이전트 워크플로우 요청하기

다음과 같이 Claude에게 병렬 작업을 요청할 수 있습니다:

```
린터와 테스트 스위트를 병렬로 실행해줘.
```

```
별도의 에이전트를 사용하여 세 가지 경쟁 라이브러리가 이 문제를 어떻게 처리하는지 조사한 다음
결과를 요약해줘.
```

```
에이전트가 이 코드 변경의 보안 영향을 검토하는 동안
너는 기능 구현을 계속해줘.
```

```
다음 기능을 구현하는 동안 백그라운드에서 통합 테스트를 실행해줘.
```

```
격리된 워크트리에서 이 기능을 구현해줘. 병합 전에 변경 사항을 검토할 수 있도록.
```

<Tip>
Claude에게 에이전트를 병렬로 실행하도록 요청하면, 하나의 메시지에 여러 Agent 도구 호출을 포함하여 모두 동시에 실행합니다.
</Tip>

## 포그라운드 vs. 백그라운드 에이전트

기본적으로 에이전트는 **포그라운드**에서 실행되며, Claude는 완료를 기다린 후 계속합니다. 포그라운드 에이전트는 Claude가 후속 작업을 진행하기 위해 결과가 필요한 시나리오에 적합합니다.

백그라운드 에이전트는 비동기로 실행됩니다. Claude가 에이전트를 실행하고 독립적으로 다른 작업을 계속하며, 완료 시 알림을 받습니다. 백그라운드 에이전트는 대화 옆의 작업 패널에 표시되어 진행 상황을 보여주고 필요한 경우 취소할 수 있습니다.

<Note>
백그라운드 에이전트는 메인 대화 옆의 작업 패널에 표시됩니다.
</Note>

포그라운드 에이전트는 완료될 때까지 진행을 차단하고, 백그라운드 에이전트는 즉각적인 결과가 필요하지 않은 병렬 워크플로우 개발을 가능하게 합니다.

## 코디네이터 모드

코디네이터 모드는 Claude를 오케스트레이터로 배치하여 구현 작업을 서브에이전트에게 위임하면서 계획, 라우팅, 종합에 집중합니다. 이 접근 방식은 Claude가 전체 워크플로우 오케스트레이션을 관리하면서 서브에이전트가 실제 구현을 수행하기를 원하는 광범위한 작업에 유용합니다.

이 모드는 주로 멀티 에이전트 팀 구성 내에서 작동합니다. 표준 단일 사용자 세션에서 Claude는 위임이 워크플로우에 도움이 되는 시점을 자율적으로 판단합니다.

## 에이전트 메모리와 컨텍스트 격리

각 서브에이전트는 **깨끗한 컨텍스트 윈도우**로 시작합니다. 부모 Claude가 에이전트의 프롬프트를 통해 완전한 작업 설명과 관련 배경을 제공합니다. 에이전트는 부모 대화 기록을 자동으로 상속하지 않습니다.

이것의 의미:
- 에이전트는 독립적으로 작동하며 부모 대화에 접근할 수 없습니다
- 부모는 에이전트 성공을 위해 프롬프트에 충분한 컨텍스트를 제공해야 합니다
- 에이전트 결과는 턴별로 스트리밍되지 않고 단일 응답으로 반환됩니다

### 영구 에이전트 메모리

특정 에이전트 유형은 마크다운 파일을 통해 호출 간 메모리를 유지합니다. 저장 위치:

- **사용자 범위**: `~/.claude/agent-memory/<agent-type>/MEMORY.md` (프로젝트 간 공유)
- **프로젝트 범위**: `.claude/agent-memory/<agent-type>/MEMORY.md` (팀 공유)
- **로컬 범위**: `.claude/agent-memory-local/<agent-type>/MEMORY.md` (로컬 전용, 커밋되지 않음)

설정된 경우, 에이전트는 이 파일을 읽고 쓰면서 세션 간에 정보를 유지하여 시간이 지남에 따라 사용자 선호도를 학습할 수 있습니다.

## 워크트리 격리

`isolation: "worktree"`를 설정하면 에이전트에게 자체 격리된 git 워크트리, 즉 별도의 저장소 복사본이 제공됩니다. 에이전트의 수정 사항은 병합하기 전까지 작업 디렉토리에 영향을 주지 않습니다.

에이전트가 변경을 수행하면 워크트리 경로와 브랜치가 결과에 포함되어 검토할 수 있습니다. 변경이 없으면 워크트리가 자동으로 정리됩니다.

## 서브에이전트 진행 상황 모니터링

에이전트 실행 중 확인할 수 있는 내용:

- 진행 표시에서 에이전트 설명과 경과 시간
- 에이전트가 실행하는 도구 호출 (트랜스크립트 모드 Ctrl+O에서 확인 가능)
- 완료 시 알림

백그라운드 에이전트는 작업 패널에 진행 상황을 표시합니다. Claude는 결과 보고 시 발견 사항을 요약합니다.

<Warning>
실행 중인 백그라운드 에이전트의 출력 파일을 직접 확인하도록 Claude에게 요청하지 마세요. Claude는 완료 알림을 자동으로 수신하며, 수동 폴링은 Claude의 컨텍스트에 불필요한 에이전트 도구 노이즈를 유발합니다.
</Warning>

## 효과적인 에이전트 프롬프트 작성

서브에이전트는 부모 대화 컨텍스트 없이 시작합니다. 자체 완결적인 브리프에는 다음이 포함되어야 합니다:

- 에이전트의 목표와 이유
- 관련 파일 경로, 함수 이름 또는 데이터
- 예상되는 보고 형식, 길이, 답해야 할 특정 질문
- 범위 제약 조건 (에이전트가 하지 *말아야 할* 것)
- 이전에 시도했거나 배제한 접근 방식

**약한 프롬프트 예시**: "조사 결과를 바탕으로 버그를 수정해줘."

**더 나은 프롬프트 예시**: "src/services/user.ts:247에 있는 UserService.getProfile()의 null 참조 버그를 수정해줘. 이 버그는 사용자에게 연결된 프로필 레코드가 없을 때 발생합니다. getProfile()이 profile이 null인지 확인하지 않고 profile.preferences를 호출합니다. null 체크를 추가하고 기본 preferences 객체 { theme: 'light', notifications: true }를 반환해줘. 수정 후 npm test를 실행하여 기존 테스트가 통과하는지 확인해줘."

## 제한 사항과 안전

- 서브에이전트는 자체 권한 모드로 작동하며, 기본값은 `acceptEdits`입니다
- 권한 규칙에서 `Agent(AgentName)` 구문을 사용하여 에이전트를 거부할 수 있습니다
- 백그라운드 에이전트는 부모의 중단 컨트롤러에 연결되지 않습니다. Escape를 누르면 부모 턴은 취소되지만 백그라운드 에이전트는 취소되지 않습니다. 명시적 취소를 위해서는 작업 패널을 사용하세요
- 서브에이전트는 다른 팀원을 생성할 수 없습니다 (플랫 팀 구성). Agent 도구를 통해 추가 서브에이전트를 생성할 수는 있습니다
- 부모 컨텍스트를 상속하는 Fork 에이전트는 자체적으로 fork할 수 없습니다. Claude가 재귀적 forking을 방지합니다
- 에이전트 결과는 부모에게 반환되기 전에 100,000자로 제한됩니다
