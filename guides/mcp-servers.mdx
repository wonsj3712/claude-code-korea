## 서버 추가하기

### CLI를 통한 추가

MCP 서버를 추가하는 가장 기본적인 방법은 `claude mcp add` CLI 명령을 사용하거나 설정 파일을 직접 편집하는 것입니다.

```bash
claude mcp add <name> -- <command> [args...]
```

예시:

```bash
claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem /tmp
```

프로젝트 범위로 추가:

```bash
claude mcp add --scope project filesystem -- npx -y @modelcontextprotocol/server-filesystem /tmp
```

사용자 범위로 추가:

```bash
claude mcp add --scope user my-db -- npx -y @my-org/mcp-server-postgres
```

### `--mcp-config` 플래그를 통한 추가

```bash
claude --mcp-config ./my-mcp-config.json
```

## 설정 파일 형식

MCP 서버 설정은 최상위 `mcpServers` 키가 있는 JSON을 사용합니다.

<Tabs>
  <Tab title="Stdio (로컬 프로세스)">
    대부분의 MCP 서버는 stdin/stdout을 통해 통신하는 로컬 서브프로세스로 실행됩니다.

    ```json
    {
      "mcpServers": {
        "filesystem": {
          "command": "npx",
          "args": ["-y", "@modelcontextprotocol/server-filesystem", "/home/user/projects"],
          "env": {
            "NODE_ENV": "production"
          }
        }
      }
    }
    ```
  </Tab>

  <Tab title="HTTP (원격 서버)">
    HTTP 엔드포인트로 호스팅되는 서버의 경우:

    ```json
    {
      "mcpServers": {
        "my-api": {
          "type": "http",
          "url": "https://mcp.example.com/v1",
          "headers": {
            "Authorization": "Bearer $MY_API_TOKEN"
          }
        }
      }
    }
    ```
  </Tab>

  <Tab title="SSE (서버 전송 이벤트)">
    SSE 전송을 사용하는 서버의 경우:

    ```json
    {
      "mcpServers": {
        "events-server": {
          "type": "sse",
          "url": "https://mcp.example.com/sse"
        }
      }
    }
    ```
  </Tab>
</Tabs>

### 환경 변수 확장

`command`, `args`, `url`, `headers`의 값은 `$VAR` 및 `${VAR}` 구문을 지원합니다. 변수는 시작 시 프로세스 환경에서 확장됩니다.

## 설정 범위

| 범위    | 위치                              | 사용 사례                        |
| ------- | --------------------------------- | -------------------------------- |
| project | 현재 디렉토리의 `.mcp.json`       | 팀 공유 서버 설정                |
| user    | `~/.claude.json` (전역 설정)      | 모든 곳에서 사용하는 개인 서버    |
| local   | `.claude/settings.local.json`     | 프로젝트별 개인 오버라이드        |

로컬 설정이 프로젝트 설정보다 우선하며, 프로젝트 설정이 사용자 설정보다 우선합니다.

## 서버 관리

### 활성화 및 비활성화

```
/mcp enable <server-name>
/mcp disable <server-name>
/mcp enable all
/mcp disable all
```

### 서버 재연결

```
/mcp reconnect <server-name>
```

### 서버 상태 확인

```
/mcp
```

`/mcp` 명령을 인수 없이 실행하면 모든 서버와 그 상태를 표시합니다. 가능한 상태값은 다음과 같습니다:

- **connected** — 연결됨
- **pending** — 대기 중
- **failed** — 실패
- **needs-auth** — 인증 필요
- **disabled** — 비활성화됨

## MCP 도구 호출 승인

Claude Code는 MCP 도구를 호출하기 전에 권한 프롬프트를 표시합니다. 프롬프트에는 도구 이름과 입력 인수가 표시되어 실행 전에 Claude가 수행할 작업을 검토할 수 있습니다.

선택 옵션:
- **한 번 허용** — 특정 호출만 승인
- **항상 허용** — 현재 세션에서 이 도구에 대한 모든 호출 승인
- **거부** — 호출 차단

자동 모드(`--allowedTools`)에서는 MCP 도구의 전체 이름(형식: `mcp__<server-name>__<tool-name>`)을 허용 도구 목록에 포함시켜 사전 승인할 수 있습니다.

## 예제: 파일시스템 서버

CLI를 통해 서버를 추가합니다:

```bash
claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem /home/user/projects
```

연결 상태를 확인한 후 Claude는 `mcp__filesystem__read_file` 및 `mcp__filesystem__write_file` 도구를 사용하여 `/home/user/projects`의 파일을 읽고 쓸 수 있습니다.

## 예제: 데이터베이스 서버

```json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "POSTGRES_CONNECTION_STRING": "$DATABASE_URL"
      }
    }
  }
}
```

## 공식 MCP 레지스트리

MCP 서버 레지스트리는 [modelcontextprotocol.io](https://modelcontextprotocol.io)에서 관리되며, 데이터베이스, 생산성 도구, 클라우드 제공업체를 위한 서버를 제공합니다.

## 문제 해결

<AccordionGroup>
  <Accordion title="서버가 'failed'로 표시됨">
    명령어의 사용 가능 여부를 확인하고, 수동으로 명령을 실행해 보고, 환경 변수를 검증하세요. 상세 로그를 위해 `claude --debug`를 사용하세요.
  </Accordion>

  <Accordion title="MCP 도구가 표시되지 않음">
    연결되었지만 인증되지 않은 서버는 도구를 노출하지 않습니다. **needs-auth** 상태를 확인하고 OAuth 인증 플로우를 완료하세요.
  </Accordion>

  <Accordion title="환경 변수가 확장되지 않음">
    변수는 시작 시 프로세스 환경에서 확장됩니다. Claude를 실행하기 전에 동일한 터미널에서 `echo $YOUR_VAR`을 사용하여 사용 가능 여부를 확인하세요.
  </Accordion>

  <Accordion title="Windows: npx 시작 실패">
    설정에서 `cmd /c`로 npx를 래핑하세요:

    ```json
    {
      "mcpServers": {
        "my-server": {
          "command": "cmd",
          "args": ["/c", "npx", "-y", "@my-org/mcp-server"]
        }
      }
    }
    ```
  </Accordion>

  <Accordion title="서버가 로컬에서는 작동하지만 CI에서 작동하지 않음">
    `--mcp-config` 플래그를 명시적으로 사용하고 CI에서 환경 변수가 설정되어 있는지 확인하세요. CI PATH에서 명령어를 사용할 수 있는지 검증하세요.
  </Accordion>
</AccordionGroup>
