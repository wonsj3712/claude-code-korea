Claude Code는 여러 인증 방법을 지원하여 Anthropic API와 클라우드 제공자 연동에 유연하게 접근할 수 있습니다.

## 인증 방법

### Claude.ai OAuth (기본)

기본 인증 방법입니다. 터미널에서 `claude`를 실행하면 claude.ai 계정을 통해 인증하는 OAuth 플로우가 시작됩니다.

```bash
claude
```

OAuth 토큰은 만료 전에 자동으로 갱신됩니다. 사용자 개입 없이 시스템이 요청 전에 토큰 유효성을 확인하고, 401 응답이 발생하면 즉시 갱신을 수행합니다.

### API 키 인증

두 가지 방법으로 API 키를 제공할 수 있습니다:

**환경 변수 사용:**

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
claude
```

**apiKeyHelper 사용:**

`~/.claude/settings.json`에서 API 키를 출력하는 스크립트를 구성할 수 있습니다. 정적 API 키를 읽는 대신 동적으로 자격 증명을 가져옵니다.

```json
{
  "apiKeyHelper": "/usr/local/bin/get-claude-key.sh"
}
```

### AWS Bedrock

Amazon Bedrock을 통해 접근하려면 `CLAUDE_CODE_USE_BEDROCK=1`을 설정하고 표준 AWS 자격 증명을 구성해야 합니다.

```bash
export CLAUDE_CODE_USE_BEDROCK=1
claude
```

AWS 자격 증명은 다음 방법으로 구성할 수 있습니다:
- 자격 증명 파일 (`~/.aws/credentials`)
- 환경 변수 (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`)
- IAM 역할
- SSO

자격 증명 만료를 처리하기 위한 추가 구성 옵션:

**awsAuthRefresh:** 자격 증명이 만료되기 전에 자동으로 갱신하는 명령을 구성합니다.

```json
{
  "awsAuthRefresh": "aws sso login --profile my-profile"
}
```

**awsCredentialExport:** 외부 소스에서 자격 증명을 내보내는 명령을 구성합니다.

```json
{
  "awsCredentialExport": "my-credential-helper export"
}
```

### GCP Vertex AI

Google Cloud의 Vertex AI를 사용하려면 `CLAUDE_CODE_USE_VERTEX=1`을 설정하고 Application Default Credentials를 구성해야 합니다.

```bash
export CLAUDE_CODE_USE_VERTEX=1
export ANTHROPIC_VERTEX_PROJECT_ID="my-gcp-project"
claude
```

자격 증명 설정 방법:
- 대화형 로그인 (`gcloud auth application-default login`)
- 서비스 계정 키
- Workload Identity

자동 토큰 갱신을 위한 구성:

```json
{
  "gcpAuthRefresh": "gcloud auth application-default login --quiet"
}
```

## 계정 관리

### 계정 전환

`/login`을 사용하여 새로운 OAuth 플로우를 시작하고 다른 계정으로 전환할 수 있습니다:

```
/login
```

### 로그아웃

`/logout`을 사용하여 저장된 자격 증명을 제거할 수 있습니다:

```
/logout
```

## 인증 우선순위

여러 인증 소스가 구성되어 있을 때, Claude Code는 다음 순서로 확인합니다:

1. `ANTHROPIC_AUTH_TOKEN`
2. `CLAUDE_CODE_OAUTH_TOKEN`
3. 파일 디스크립터의 OAuth 토큰
4. `apiKeyHelper`
5. 저장된 OAuth 토큰
6. `ANTHROPIC_API_KEY`

목록에서 위에 있는 소스가 아래 소스보다 우선합니다. 예를 들어 `ANTHROPIC_AUTH_TOKEN`이 설정되어 있으면 `ANTHROPIC_API_KEY`가 있더라도 `ANTHROPIC_AUTH_TOKEN`이 사용됩니다.
