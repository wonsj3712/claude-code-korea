Claude Code는 Claude를 기반으로 한 터미널 기반 AI 에이전트입니다. 파일 시스템, 셸, 도구에 직접 접근할 수 있어, 작업을 자연어로 설명하면 Claude가 구현을 처음부터 끝까지 처리합니다 — 복사/붙여넣기도, 컨텍스트 전환도 필요 없습니다.

## 권한 시스템

Claude Code의 모든 도구 사용은 권한 확인을 거칩니다. Claude에게 얼마나 많은 자율성을 부여할지 직접 제어할 수 있습니다.

| 모드                | 동작                                                                                                                             |
| ------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `default`           | Claude가 셸 명령어 실행과 편집 전에 확인을 요청합니다. 각 작업을 승인하거나 거부할 수 있습니다.                                     |
| `acceptEdits`       | 파일 편집이 자동으로 적용됩니다. 셸 명령어는 여전히 승인이 필요합니다.                                                             |
| `plan`              | Claude가 계획을 작성하고 실행 전에 승인을 요청합니다. 대규모 변경 사항을 실행 전에 검토하기에 좋습니다.                               |
| `bypassPermissions` | 모든 작업이 확인 없이 실행됩니다. 샌드박스 환경의 자동화 파이프라인 전용으로, 대화형 사용에는 적합하지 않습니다.                       |

<Warning>
  `bypassPermissions` 모드는 모든 확인 프롬프트를 비활성화합니다. Claude가 작업 범위 외의 시스템에 영향을 줄 수 없는 격리된 환경(Docker 컨테이너, CI 샌드박스)에서만 사용하세요.
</Warning>

Claude를 시작할 때 `--permission-mode` 플래그로 권한 모드를 설정하거나, `/permissions` 명령어로 변경할 수 있습니다. `/permissions`에서는 특정 도구에 대한 허용/거부 규칙도 관리할 수 있습니다:

```bash
claude --permission-mode acceptEdits
```

## CLAUDE.md 메모리 시스템

Claude Code는 매 세션 시작 시 저장소의 `CLAUDE.md` 파일을 읽습니다. 이 파일에 프로젝트별 지식 — 빌드 명령어, 코딩 컨벤션, 아키텍처 노트, 필수 환경 변수 등 — 을 기록해두면, Claude가 매번 처음부터 다시 파악할 필요가 없습니다.

세 가지 범위가 있습니다:

* **프로젝트** (저장소 루트의 `CLAUDE.md`) — 팀 전체가 공유하며, 소스 제어에 커밋합니다.
* **개인** (저장소 루트의 `CLAUDE.local.md`) — 이 프로젝트에 대한 개인 설정으로, gitignore에 추가합니다.
* **하위 디렉토리** (하위 디렉토리 내의 `CLAUDE.md`) — Claude가 해당 디렉토리에서 작업할 때 자동으로 로드됩니다. 독립적인 모듈이 있는 모노레포에 유용합니다.

Claude Code 세션에서 `/init`을 실행하면 프로젝트용 `CLAUDE.md`를 자동으로 생성합니다. Claude가 코드베이스를 분석하고 가장 중요한 명령어와 컨텍스트가 담긴 파일을 생성합니다.

```bash
# Claude Code 세션 내에서
/init
```

<Tip>
  `CLAUDE.md`는 간결하게 유지하세요. 모든 줄은 "이 내용을 제거하면 Claude가 실수할까?"라는 테스트를 통과해야 합니다. 그렇지 않다면 삭제하세요.
</Tip>

## 인증

Claude Code는 두 가지 방식으로 인증합니다:

1. **OAuth (권장)** — `claude.ai`에서 Anthropic 계정으로 로그인합니다. `claude`를 처음 실행하고 브라우저 프롬프트를 따라하세요.
2. **API 키** — `ANTHROPIC_API_KEY` 환경 변수를 설정합니다. CI 파이프라인 및 비대화형 세션에 유용합니다.

전체 우선순위와 클라우드 제공자 옵션(AWS Bedrock, GCP Vertex AI)은 [인증](/guides/authentication) 문서를 참조하세요.
